# CS-230-Software-Design-Document
Portfolio submission for CS 230: Operating Platforms, including The Gaming Room software design document and reflection.

## CS 230 Module Eight Journal Reflection

For this project, The Gaming Room was the client, and they wanted to take their Android game, Draw It or Lose It, and expand it into a web-based game that could work across different platforms. They needed the software to support multiple games, teams, and players while making sure each game had a unique name and could run in a way that was reliable, scalable, and secure. I think one thing I did well in the design document was organizing the client’s needs into a clear plan instead of just jumping straight into code. Working through the document helped because it made me slow down and think about how the application should be structured, how the classes connected to each other, and why certain design patterns made sense before actually building anything. If I could go back and revise one part, I would probably clean up the recommendations section and make it more specific, especially when explaining why certain platforms or design choices would be better for the client long term. I tried to interpret the user’s needs by focusing on what the game actually had to do for the people using it, not just what the code needed to do. That matters because software can technically run and still not be useful if it does not match what the client and users actually need. My approach was to start with the requirements, break the system into smaller pieces, compare platform options, and then use object-oriented design to guide the structure. In the future, I would use the same kind of process because having a design plan first makes the coding part feel a lot less random and helps keep the project focused.
